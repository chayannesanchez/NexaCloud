# Soporte

## Cómo pedir ayuda

Para reportar errores o solicitar mejoras, crea un issue con esta información:

- Descripción del problema.
- Pasos para reproducirlo.
- Resultado esperado.
- Resultado actual.
- Capturas o logs si aplica.
- Ambiente: `dev`, `prod` u otro.

## Logs útiles

Incluye logs de CloudWatch de la Lambda relacionada, evitando datos sensibles.

## Categorías sugeridas de issues

- `bug`: error funcional.
- `docs`: mejora de documentación.
- `infra`: cambios de Terraform/AWS.
- `frontend`: cambios visuales o JS.
- `backend`: cambios de Lambdas.
- `security`: riesgo o mejora de seguridad.
